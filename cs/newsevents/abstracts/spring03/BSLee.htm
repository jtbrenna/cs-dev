Abstract

 

Web caching is a technique for accelerating client applications and reducing the loads on web servers and the network by caching web server data in a local cache server. Unlike traditional caching, web caching relaxes cache consistency requirement by allowing for delayed cache refreshment to achieve better performance while tolerating some inconsistency. This relaxed requirement has introduced time-to-live (TTL), defined as the time interval in which a cached data item is expected to be valid. Typically a cache server estimates the TTL if the web server does not provide it. Two estimation methods are well known: the fixed TTL method and the heuristic method. Unfortunately, they have no theoretical background and are not intuitive.

 

We propose a new method based on the notion of an update-risk, which is defined as the probability of the original data being updated within the estimated TTL. This method uses a probabilistic approach based on the Poisson process. Users are asked to provide the value of an update risk, and the cache server calculates TTL based on it. Experiments show that the update risk measured from a real cache server log is almost the same as the update risk used to estimate the TTL. We also demonstrate that our method is more intuitive and general than the methods mentioned above.
